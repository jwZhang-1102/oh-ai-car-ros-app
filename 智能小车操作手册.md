# 智检哨兵 · 操作手册（精简版）

> 面向演示与验收的简明说明。  
> 小车默认 IP：`10.147.13.194`  
> Jetson 工作目录：`~/Rosmaster-App/rosmaster/`  
> Windows 项目目录：`D:\oh-ai-car-ros-app`

---

## 1. 系统概述

本系统在亚博 ROS2 小车（Jetson）上扩展了 **视觉巡检** 与 **多端人机协同**，并配套 OpenHarmony App。

| 能力 | 说明 |
|------|------|
| 自主导航 | Docker 内 ROS2 导航：到目标点自动行驶 |
| 智能巡检 | YOLO 识别异常（如瓶子），报警、截图，App 可查历史 |
| 手机遥控 | 鸿蒙 App：摇杆 / 方向键控车 + 视频直播 |
| 方向盘驾驶 | PC 端罗技 G29，经 TCP 控车 |
| 手势控车 | 摄像头识别手势，控制小车运动 |
| 音乐伴舞 | 播放音乐并执行编排动作（展示用） |

**工作模式说明（验收口径）**

- **巡防模式**：导航容器 + 巡检同时运行，手机 App 用「巡检模式」查看告警即可。  
- **遥控模式**：需先停止导航容器，再启动 `ros/run`，然后用 App 或 G29 控车。  
- 当前实现为 **模式切换**，不是「导航过程中的无缝热接管」。

## 2. 网络端口

| 端口 | 协议 | 用途 |
|------|------|------|
| **6000** | TCP | App / G29 控车 |
| **6500** | HTTP | 默认摄像头视频（`/index2`、`/video_feed`） |
| **6501** | HTTP | 可选低延迟视频（需单独启动，见 5.3） |
| **6700** | HTTP | 巡检服务（告警列表、截图） |

手机、电脑与小车须在同一局域网。浏览器地址**必须带端口**，不要只开 `http://10.147.13.194`（80 端口无服务，会显示「拒绝连接」）。

```text
http://10.147.13.194:6500/index2
http://10.147.13.194:6500/video_feed
```

Jetson 上自检：

```bash
ss -tlnp | grep -E '6000|6500|6501|6700'
curl -sf http://127.0.0.1:6700/health
```

## 3. 资源互斥（操作前必读）

下列组合 **不要同时开启**：

| 互斥项 | 原因 |
|--------|------|
| `ros/run` 与 Docker 导航（n1） | 争用底盘串口 |
| 6500 视频与 YOLO 巡检 | 争用 USB 摄像头 |
| `music.py` 与 `ros/run` / 导航 | 争用串口 |
| App 与 G29 同时控车 | 共用 TCP 6000 |

| 推荐组合 | 说明 |
|----------|------|
| 导航 n1/n2/n3 + 巡检后台 | 主演示：边走边检 |
| 巡防阶段 App 仅用巡检模式 | 只访问 6700，不连 6000 |
| App / G29 | 先停导航容器，再 `ros/run` |
| 音乐伴舞 | 单独演示，停导航与遥控 |

## 4. 推荐演示流程

```text
① 自主巡防（主场景）
   启动巡检后台 → 启动导航 → RViz 设起点与目标
   → 发现瓶子报警短停 → 继续航点
   → App「巡检模式」查看历史告警

② 人工控车（模式切换）
   停止导航容器 → ros/run → App 遥控 或 G29 方向盘

③（可选）手势控车
   s 开容器 → 一终端 d + n1 → 另一终端 d 后执行 gesture_control_ros2.py

④（可选）音乐伴舞
   停止导航与 ros/run → 运行 music.py
```

## 5. Jetson 常用操作

以下均在 Jetson SSH 终端、目录 `~/Rosmaster-App/rosmaster/` 下执行（容器内命令另行注明）。

### 5.1 主场景：导航 + 巡检

```bash
cd ~/Rosmaster-App/rosmaster

# 使用最新地图时，首次建议同步到容器
bash sync_nav_map_to_docker.sh

# 1) 巡检（后台）
bash start_patrol_host.sh --bg（建议完全启动后关闭该命令行）
curl -sf http://127.0.0.1:6700/health
```

**2) 启动自动导航（宿主机 + 容器，与使用手册一致）**

| 步骤 | 操作 |
|------|------|
| ① | 宿主机：`s` 启动导航容器（或 `docker start <容器ID>`） |
| ② | 新开 **3 个** Terminal，分别输入 `d` 进入同一容器 |
| ③ | 终端 A：`n1`（导航基础：雷达 + 底盘），等待约十余秒就绪 |
| ④ | 终端 B：`n2`（RViz 展示） |
| ⑤ | 终端 C：`n3`（DWA 导航；也可用 `n4` 换 TEB） |
| ⑥ | 在 RViz：**2D Pose Estimate** 设初始位姿（必做） |
| ⑦ | 再设 **2D Goal Pose**，小车自主行驶 |

> `s` / `t` / `d` 以及 `n1`～`n4` 为环境内快捷指令，定义见使用手册与容器 `~/.bashrc`。容器 ID 以 `docker ps -a` 为准。

停止巡检：

```bash
bash stop_patrol_host.sh
```

导航结束：各终端 `Ctrl+C` 结束节点后 `exit` 退出容器，宿主机执行 `t`（或 `docker stop <容器ID>`）停容器。

### 5.2 切换到遥控（Jetson 侧准备）

App 与 G29 **都**需要先在 Jetson 上进入遥控服务；二者互斥，不要同时控车。

```bash
t                            # 或 docker stop <容器ID>，停导航容器
bash stop_patrol_host.sh     # 需要 6500 视频时释放摄像头
ros/run                      # 提供 TCP 6000（及视频）
ss -tlnp | grep -E '6000|6500'
```

| 遥控方式 | 在哪里操作 | 说明 |
|----------|------------|------|
| **鸿蒙 App** | 手机 | 见第 7 节，无需 PC 脚本 |
| **罗技 G29** | **Windows 电脑** | 方向盘接 PC；还需安装驱动并在本机运行控车脚本，见第 6 节 |

停止遥控：在 `ros/run` 终端按 `Ctrl+C`。

### 5.3 低延迟视频（可选，端口 6501）

默认 `ros/run` 的 **6500** 即可看直播；**6501** 为可选低延迟推流（方向盘演示时更合适）。  
未启动 6501 时访问该端口会 **拒绝连接**——属正常现象。

**启动顺序（重要）**

1. 停掉占用摄像头的进程：巡检、`ros/run`（另终端 `Ctrl+C`）  
2. 启动 6501  
3. 再开终端启动 `ros/run`（控车 6000）

```bash
cd ~/Rosmaster-App/rosmaster
bash stop_patrol_host.sh 2>/dev/null || true
# 若本机正跑着 ros/run，先在其终端 Ctrl+C

bash start_low_latency_video.sh        # 流畅 320×240
# bash start_low_latency_video.sh --hd # 更清晰 640×480
# bash start_low_latency_video.sh --hd --bg  # 后台运行

# 另开终端再启控车
ros/run
```

**本机 / 手机浏览器（必须写端口 6501）**

```text
http://10.147.13.194:6501/
http://10.147.13.194:6501/index2
http://10.147.13.194:6501/video_feed
```

Jetson 上确认已监听：

```bash
ss -tlnp | grep 6501
curl -I http://127.0.0.1:6501/video_feed
```

| 对比 | 6500（默认） | 6501（低延迟） |
|------|--------------|----------------|
| 谁提供 | 一般随 `ros/run` / 摄像头服务 | 需 `start_low_latency_video.sh` |
| 用途 | 日常预览 | G29 / 驾驶，延迟更低 |
| 未启动时 | — | 浏览器「拒绝连接」 |

### 5.4 手势控车（容器内）

需导航容器已开，且 **n1**（底盘/雷达基础）在跑。建议先取消导航 Goal（或不要开 n3），避免与手势抢 `/cmd_vel`。

| 步骤 | 操作 |
|------|------|
| ① | 宿主机：`s` 启动导航容器 |
| ② | 终端 A：`d` 进入容器 → 执行 `n1`，保持运行 |
| ③ | **再开一个** Terminal：`d` 进入同一容器 |
| ④ | 在终端 B 执行：`python3 /root/gesture_control_ros2.py` |

```bash
# 宿主机
s

# 终端 A（进入容器后）
d
n1

# 终端 B（另开窗口，再进入容器）
d
python3 /root/gesture_control_ros2.py
```

### 5.5 音乐伴舞

```bash
# 先停止 ros/run 与导航容器
amixer -c 0 set PCM 90% unmute
python3 music.py
```

### 5.6 导航相关快捷指令一览

| 位置 | 指令 | 作用 |
|------|------|------|
| 宿主机 | **s** / **t** / **d** | 启动 / 停止 / 进入导航容器 |
| 容器内 | **n1** | 导航基础节点 |
| 容器内 | **n2** | RViz 展示 |
| 容器内 | **n3** | DWA 导航（常用） |
| 容器内 | **n4** | TEB 导航（可选） |

详细说明见 [智能小车使用手册.md](./智能小车使用手册.md) 第 3.7 节。

## 6. Windows（PC）罗技 G29 方向盘

G29 **不是**在 Jetson 上启动，而是：方向盘 USB 接 Windows 笔记本 → 本机运行脚本 → 经 WiFi 连小车 `6000` 端口。

### 6.1 软件准备（首次）

1. **安装罗技驱动 / G HUB**  
   - 下载安装 [Logitech G HUB](https://www.logitechg.com/software/g-hub)（或官方 G29 驱动）  
   - 插上 G29 后，在设备管理器中能识别到游戏控制器  
2. **Python 依赖**（本机）：`pip install pygame`  
3. **Jetson** 已按第 5.2 节执行 `ros/run`，且 `6000` 在监听  

> 运行控车脚本前，建议在系统托盘 **退出 G HUB**（避免独占方向盘导致读不到轴）。驱动仍需事先装好。

### 6.2 启动控车（本机）

```cmd
cd D:\oh-ai-car-ros-app\tools
python logitech_g29_drive.py --backend pygame --mode pedal --ip 10.147.13.194
```

若 pygame 读不到踏板/方向盘，可改用：

```cmd
python logitech_g29_drive.py --backend winmm --mode pedal --ip 10.147.13.194
```

| 踏板 / 操作 | 动作 |
|-------------|------|
| 油门 | 前进 |
| 离合 | 后退 |
| 刹车 | 停止 |
| 未踩踏板转方向盘 | 原地左右旋转 |

更细的标定与参数见 [tools/README.md](./tools/README.md)。

## 7. OpenHarmony App

| 场景 | 操作 |
|------|------|
| 巡检查看 | 网络页填写 IP → **保存并进入（巡检模式）** → 智能巡检 |

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| App / G29 无法控车 | 确认已停导航容器，且 `ros/run` 使 6000 在听 |
| 视频黑屏 | 确认 6500（或 6501）已启动；巡检占摄像头时先停巡检 |
| 导航不走 | RViz 是否已做 2D Pose Estimate；目标点是否合理 |
| 音乐无动作或无声音 | 确认串口未被占用；检查音量（PCM）是否打开 |
