# 智能小车操作指令

> 汇总 **Windows 本地（PC）** 与 **Jetson 小车** 上操控、巡检、导航相关命令。  
> 默认小车 IP：`10.147.13.194`  
> Jetson 工作目录：`~/Rosmaster-App/rosmaster/`  
> PC 项目目录：`D:\oh-ai-car-ros-app`

---

## 一、端口与资源（必记）

| 端口 | 用途 | 依赖服务 |
|------|------|----------|
| **6000** | TCP 遥控（App / G29） | Jetson `ros/run` |
| **6500** | HTTP 视频 MJPEG | 摄像头服务 |
| **6700** | 巡检 HTTP（事件/截图） | `patrol_server.py` |

```bash
# Jetson 检查端口
ss -tlnp | grep -E '6000|6500|6700'
curl -sf http://127.0.0.1:6700/health
curl -sf http://127.0.0.1:6500/index2 -I
```

## 二、什么不能同时开

| ❌ 禁止组合 | 原因 |
|------------|------|
| App/G29 遥控（6000）+ Docker 导航 n3 | 抢底盘控制 |
| `ros/run` + Docker n1 | 抢底盘串口 |
| 6500 视频 + YOLO 巡检 | 抢 USB 摄像头 |
| App 遥控 + G29 同时发指令 | 共用 TCP 6000 |

| ✅ 可并行 | 说明 |
|----------|------|
| Docker n1+n3 + 巡检 6700 | 用 `start_patrol_host.sh --bg`（默认 on-demand 危险区） |
| G29 遥控 + 巡检 6700 | 需 `ros/run`；与 YOLO 抢摄像头时勿开视频 |

## 三、Windows 本地（PC）命令

### 3.1 罗技 G29 方向盘控车

**前置**：Jetson 已执行 `ros/run`，6000 在监听。建议 **完全退出 G HUB**。

```cmd
cd D:\oh-ai-car-ros-app\tools
pip install pygame
```

| 用途 | 命令 |
|------|------|
| 标定踏板/方向盘 | `python logitech_g29_drive.py --backend winmm --calibrate` |
| **踏板控车（推荐）** | `python logitech_g29_drive.py --backend winmm --mode pedal --ip 10.147.13.194` |
| 仅方向盘旋转 | `python logitech_g29_drive.py --backend winmm --mode rotate --max-turn 55 --ip 10.147.13.194` |
| 比例弧线行驶 | `python logitech_g29_drive.py --backend winmm --mode drive --max-speed 30 --ip 10.147.13.194` |
| 测试 TCP 旋转 | `python logitech_g29_drive.py --test-rotate --ip 10.147.13.194` |
| 打印发送帧 | 加 `--verbose` |
| 踏板不灵敏 | 加 `--pedal-threshold 0.08` |
| 列出 winmm 设备 | `python logitech_g29_drive.py --list-devices` |

**`--mode pedal` 映射（与 App 方向键一致）**

| G29（左→右） | 动作 | cmd15 |
|--------------|------|-------|
| 离合 | 后退 | dir=2 |
| 刹车 | 停止 | dir=0 |
| 油门 | 前进 | dir=1 |
| 方向盘（未踩踏板） | 左/右旋转 | dir=5/6 |

### 3.2 G29 模拟驾驶 + 低延迟视频

Yahboom 默认 `6500/video_feed` 容易积压旧帧（15fps 仍可能 **~3 秒延迟**）。改用仓库自带低延迟推流（**6501**）：

**Jetson**（顺序重要：**先停 `ros/run`**，再起 6501，**最后**再 `ros/run`）：

```bash
cd ~/Rosmaster-App/rosmaster
# 1) 另终端若已在跑 ros/run，先 Ctrl+C 停掉（会占摄像头）
bash stop_patrol_host.sh 2>/dev/null || true
bash stop_camera_server.sh 2>/dev/null || true
fuser -v /dev/video0    # 应无输出

# 2) 低延迟视频（6501）
bash start_low_latency_video.sh        # 流畅 320x240
# bash start_low_latency_video.sh --hd # 清晰 640x480
# bash start_low_latency_video.sh --bg # 后台

# 3) 另开终端再启控车（G29 需要 6000）
ros/run
```

若报 `Device or resource busy`：`ros/run` 未停或摄像头未释放，按上面顺序重来。

确认：

```bash
curl -I http://127.0.0.1:6501/video_feed
curl http://127.0.0.1:6501/health
```

**PC 看画面**（任选）：

```cmd
ffplay -f mpjpeg -fflags nobuffer -flags low_delay -framedrop http://10.147.13.194:6501/video_feed
```

浏览器：`http://10.147.13.194:6501/` 或 `/video_feed`

**PC G29**（与 3.1 相同）：

```cmd
cd D:\oh-ai-car-ros-app\tools
python logitech_g29_drive.py --backend winmm --mode pedal --ip 10.147.13.194
```

摄像头须 **MJPG**（勿用 YUYV 1080p，否则仅 ~3fps）：

```bash
v4l2-ctl -d /dev/video0 --set-fmt-video=width=320,height=240,pixelformat=MJPG
v4l2-ctl -d /dev/video0 --get-fmt-video   # 应显示 MJPG
```

**画质 / 延迟调参**（默认 320×240 q=60，偏糊但延迟低；你测得 ~1s 可接受后可上调）：

| 档位 | 分辨率 | quality | 预期 |
|------|--------|---------|------|
| 默认（流畅） | 320×240 | 60 | 延迟最低，画质一般 |
| **推荐平衡** | **640×480** | **70** | 清晰不少，延迟略增 |
| 高清 | 640×480 | 80 | 更清晰，延迟/带宽再增 |

手动启动（先 **Ctrl+C 停掉** 正在跑的 6501 和 `ros/run`，确认 `fuser /dev/video0` 无输出）：

```bash
pkill -f low_latency_mjpeg.py 2>/dev/null || true
fuser -v /dev/video0    # 应无输出
bash start_low_latency_video.sh --hd   # 推荐：一键 640x480
```

PC 全屏：

```cmd
ffplay -fs -f mpjpeg -fflags nobuffer -flags low_delay -framedrop http://10.147.13.194:6501/video_feed
```

**规律**：分辨率影响最大，quality 次之；用挥手法测，延迟仍 &lt;1.5s 即可用。

上传新脚本：

```cmd
scp jetson/low_latency_mjpeg.py jetson/start_low_latency_video.sh jetson@10.147.13.194:~/Rosmaster-App/rosmaster/
```

### 4.2 智能巡检（6700 + YOLO）


### 4.3 Docker 自主导航（一键）

**使用宿主机最新地图（如 7 月 7 日 `src/maps/yahboomcar.*`）**

n3 读的是容器内 `install/.../maps/yahboomcar.yaml`，与宿主机 `src` 不同步。启动 n1/n2/n3 **之前**在宿主机执行：

```bash
cd ~/Rosmaster-App/rosmaster
bash sync_nav_map_to_docker.sh
```

源路径默认：`/home/jetson/code/yahboomcar_ws/src/yahboomcar_nav/maps/yahboomcar.pgm`  
可改：`HOST_MAP_DIR=/你的路径 bash sync_nav_map_to_docker.sh`

同步后：**n1 → n3 → n2**，RViz 必须重新 **2D Pose Estimate**。

```bash
cd ~/Rosmaster-App/rosmaster

# 一键：docker start + 开 3 个终端 n1 → n2 → n3
bash start_nav_docker.sh

# 三个终端同时开（需自行确认 n1 就绪）
bash start_nav_docker.sh --nowait

# 无桌面 SSH 用 tmux
bash start_nav_docker.sh --tmux
tmux attach -t rosmaster_nav

# 指定容器 ID 前缀
NAV_DOCKER_CID=2169 bash start_nav_docker.sh
```

**RViz 操作（容器 n2 窗口）**

1. **2D Pose Estimate** — 设初始位姿（必须先做）
2. **2D Goal Pose** — 设目标点，小车自主走

### 4.4 Docker 容器快捷指令（宿主机 ~/.bashrc）

> 容器 ID 以你机器为准（手册示例 `2169`），先 `docker ps -a` 查看。

| 快捷指令 | 完整含义 |
|----------|----------|
| `s` | `docker start <容器ID>` 启动容器 |
| `t` | `docker stop <容器ID>` 停止容器 |
| `d` | `docker exec -it <容器ID> /bin/bash` 进入容器 |

```bash
docker ps -a
docker ps
source ~/.bashrc   # 使 s/t/d 生效
```

## 五、Docker 容器内命令

> 先 `d` 进入容器。以下 `n1/n2/...` 为容器内 `~/.bashrc` 快捷指令（autodrive 镜像）。

### 5.1 自主导航（手册 3.7）

| 快捷指令 | 完整命令 | 用途 |
|----------|----------|------|
| **n1** | `ros2 launch yahboomcar_nav laser_bringup_launch.py` | 导航基础（雷达+底盘） |
| **n2** | `ros2 launch yahboomcar_nav display_nav_launch.py` | RViz 展示 |
| **n3** | `ros2 launch yahboomcar_nav navigation_dwa_launch.py` | DWA 导航 |
| **n4** | `ros2 launch yahboomcar_nav navigation_teb_launch.py` | TEB 导航（可选） |

**推荐顺序**：`n1` → 等 15s → `n2` → `n3` → RViz 设位姿与目标点

**容器内诊断（Foxy，无 `--once`）**

```bash
ros2 topic hz /scan --window 5
ros2 topic hz /odom --window 5
timeout 5 ros2 topic echo /amcl_pose 2>&1 | head -20
timeout 8 ros2 run tf2_ros tf2_echo map base_footprint
timeout 5 ros2 topic hz /cmd_vel --window 5
ros2 lifecycle get /map_server
ros2 lifecycle get /amcl
```
